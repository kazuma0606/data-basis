# ユーザーストーリー

## ビジネスコンテキスト

**想定企業**: 中規模リテール企業（例：地方の家電量販店チェーン）

### データソース

| システム | 役割 |
|---|---|
| ECサイト | オンライン購買・閲覧ログ |
| 実店舗POS | 来店・レジデータ |
| 会員アプリ | 行動ログ・ポイント |
| 在庫管理 | 商品マスタ・在庫 |

---

## ペルソナ

**顧客A: 田中さん（35歳・男性）**
- ECで冷蔵庫を購入済み
- 店舗でたまに来るが、いつも家電売り場だけ
- アプリは入れているがほぼ通知オフ

**顧客B: 佐藤さん（28歳・女性）**
- ECでホームシアター関連を何度も閲覧、未購入
- 店舗来店なし
- 友人（会員）が最近プロジェクター購入

---

## ユーザーストーリー

```
As マーケター
I want 「冷蔵庫を買った人が3ヶ月後に何を買いがちか」を
        チャネル横断で見たい
So that 田中さんに調理家電をサジェストできる
        （明示的な興味ゼロだが、購買パターン上は高確率）
```

```
As データサイエンティスト
I want 「閲覧→未購入」の弱いシグナルを持つ顧客のクラスタを
        作りたい
So that 佐藤さんのような潜在顧客に対して
        「友人が買った」「今週セール」を組み合わせた
        トリガーベースの通知を打てる
```

```
As 店長
I want 来週来店しそうな顧客を事前に知りたい
So that スタッフ配置と在庫を最適化できる
        （来店予測 × 購買余地スコア）
```

## 「強いシグナル」と「弱いシグナル」

| 強いシグナル (explicit) | 弱いシグナル (implicit) |
|---|---|
| 商品ページを3回以上閲覧 | カテゴリページを通過 |
| カートに入れた | 滞在時間が平均より長い |
| 検索した | 似た属性の顧客が買った |
| レビューを読んだ | 購買後N日が経過した |

弱いシグナルは1つだと雑音だが、Kafkaでリアルタイムに積み重ねてスコアリングすると意味が出てくる。

### スコアリング例（田中さん・調理家電カテゴリ）

```
= 冷蔵庫購入から90日       (+30)
+ 同属性顧客の購買率        (+25)
+ 先週キッチン売り場に3分滞在 (+20)
─────────────────────────────
潜在スコア: 75 → 通知トリガー閾値超え
```

## データフロー（概要）

```
[ECログ]   ──┐
[POSデータ]  ─┼→ Kafka → スコアリングサービス(k8s) → 通知/CRM
[アプリログ] ─┘              ↓
                        S3(LocalStack) → 分析・モデル学習
```
